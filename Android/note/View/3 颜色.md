# 颜色  
## 颜色模式  
* ARGB8888：四通道高精度（32位）  
* ARGB4444：四通道低精度（16位）  
* RGB565：  屏幕默认模式（16位）  
* Alpha8:   仅有透明通道（8位）  
### 表现形式  
```  
#fff        不含透明通道，低精度  
#afff       含透明通道，低精度  
  
#f1f1f1     不含透明通道，高精度  
#aaf1f1f1   含透明通道，高精度  
```  
## 颜色混合模式  
**由Alpha通道决定**  
> 所谓的透明度，实际上不是真正的透明，而是将颜色混合，造成透明的效果  
  
### 默认情况下，当在canvas上绘制一个有Alpha通道的颜色时，最终的颜色的计算公式：  
**最终颜色 = 要绘制的颜色 + （1 - 绘制颜色的透明度）* Canvas上的颜色**  
> 这里有几点要注意的：  
> * 在计算的时候需要注意你使用的颜色模式  
> * 把各个通道上的值映射为0到1的浮点数（比如你的颜色使用的是高精度，就用该通道上的值除以255）  
> * 这个公式是计算单个通道的最终颜色  
> * 要绘制的颜色是指该通道上的颜色与Alpha乘积之后的值，比如一个颜色值是“#aaFFF181”，那么在蓝色通道上的要绘制的颜色的浮点值就是：(129/255)*(170/255);假设此时canvas上的颜色是“#FF4081”，最终的结果就是：((129/255)*(170/255) + (1 - 170/255)*(129/255))*255 = 128,也就是蓝色通道上的值最终是0x80；  
> * 同理，canvas上的颜色也是对应通道的颜色与Alpha乘积之后的值  
*这里清楚原来就行了，再说谁没事去计算这个玩意儿呢*
